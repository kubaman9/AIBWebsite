# GitHub Pages Deployment Steps

## 1. Create GitHub Repository
- Go to https://github.com/new
- Repository name: `AIBwebsite`
- Make it **Public** (required for GitHub Pages)
- Click "Create repository"

## 2. Add Remote and Push
In PowerShell, run:
```powershell
cd "c:\Users\kubam\AIB WEBSITE.2\AIBwebsite"
git remote add origin https://github.com/YOUR_USERNAME/AIBwebsite.git
git branch -M main
git push -u origin main
```

Replace `YOUR_USERNAME` with your actual GitHub username.

## 3. Update Configurations (After you know your username)
Update these files with your GitHub username:

**package.json** (line 6):
```json
"homepage": "https://YOUR_USERNAME.github.io/AIBwebsite"
```

**vite.config.js** (line 7):
```javascript
base: '/AIBwebsite/'
```

## 4. GitHub Pages Settings
- Go to your repository on GitHub
- Settings â†’ Pages
- Source: Deploy from a branch
- Branch: `gh-pages` (auto-created by GitHub Actions)
- Folder: `/ (root)`
- Click Save

## 5. Enable GitHub Actions
- Go to Actions tab in your repository
- Enable GitHub Actions if prompted
- The workflow file (.github/workflows/deploy.yml) will auto-deploy on push

## 6. Live URL
Your site will be live at:
```
https://YOUR_USERNAME.github.io/AIBwebsite
```

## Build & Test Locally
```powershell
npm run build
npm run preview
```

## Troubleshooting
- Make sure repository is PUBLIC
- Check Actions tab for deployment errors
- GitHub Pages can take 1-2 minutes to deploy
- Hard refresh browser (Ctrl+Shift+R) to see updates
